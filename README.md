# 任务数据面板

## 项目简介

这是一个使用原生 JavaScript 完成 API 交互与数据渲染的任务管理面板页面。页面通过调用后端 RESTful API 实现任务的增、查、筛选等功能，无需任何第三方框架或库。

## 完成的功能

- 页面加载后自动获取并渲染任务列表
- 点击"刷新任务"按钮重新获取数据
- 通过表单提交新任务（POST 请求）
- 按状态（全部 / 待开始 / 进行中 / 已完成）筛选任务
- 接口请求失败时显示错误提示
- 提交成功 / 失败时给予用户反馈

## 实现思路

### 1. 页面加载

页面加载后，通过 `fetch` 向 `GET /api/tasks` 发起请求，获取当前所有任务数据，并缓存到 `allTasks` 数组中，随后调用渲染函数将数据展示在 `#taskList` 列表中。

### 2. 获取和渲染数据

使用 `renderTasks()` 函数遍历任务数组，为每条任务动态创建 `<li>` 元素，包含任务标题、负责人、状态徽标和创建时间。对输出内容做了 HTML 转义以防止 XSS。

### 3. 提交任务

监听表单的 `submit` 事件，阻止默认提交行为后从表单控件中获取 `title`、`owner`、`status` 三个字段，通过 `fetch` 以 `POST` 方式发送 JSON 数据到 `/api/tasks`。提交成功后自动重置表单并刷新列表。

### 4. 状态筛选

将接口返回的完整任务列表缓存在变量 `allTasks` 中，当用户通过下拉框切换状态时，使用 `Array.filter()` 在前端过滤并重新渲染，无需再次请求接口。

### 5. 错误处理

所有 `fetch` 请求均使用 `.catch()` 捕获异常，将错误信息显示在对应的提示区域（`#formMessage` 或 `#listMessage`），并用不同颜色区分成功（绿色）与失败（红色）状态。

## 技术要点

| 类别 | 说明 |
|------|------|
| DOM 操作 | `getElementById`、`createElement`、`innerHTML` |
| 事件绑定 | `addEventListener`（click / submit / change） |
| 网络请求 | `fetch` API + `Promise` 链式调用 |
| 数据处理 | `Array.filter`、`Array.forEach` |
| 安全防护 | HTML 特殊字符转义（防 XSS） |
